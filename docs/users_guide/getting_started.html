Getting BidiChecker

Browser bookmarklet: To install the bookmarklet in your browser's Bookmarks toolbar, go to the BidiChecker Bookmarklet Installation Page. You will find two links labeled "BidiChecker LTR" and "BidiChecker RTL". Dragging each of those links to your Bookmarks toolbar will install a button which, when clicked, runs BidiChecker on the page currently in the browser. The LTR bookmarklet expects the page to have a left-to-right UI; the RTL bookmarklet is for pages with a right-to-left UI.

Source code: The BidiChecker source code can be downloaded from http://code.google.com/p/bidichecker using the following command:

svn checkout http://bidichecker.googlecode.com/svn/trunk/ bidichecker

Source code and unit tests can be found in the src/ subdirectory. This user's guide is in the docs/users_guide subdirectory, with the complete API documentation in docs/jsdoc/. A sample application, along with demonstrations of how BidiChecker can be used to check for bidi errors, can be found in samples/reviews/.

The lib/ subdirectory contains a self-contained, deployable installation of BidiChecker. If you copy this directory to a web server, you can easily run BidiChecker either from automated unit tests or via the browser bookmarklet. The directory contains:

  1. a single-file, precompiled version of BidiChecker, named bidichecker_packaged.js. Loading this one file from the web page to be tested is the usual way to use BidiChecker (you only need BidiChecker's source code if you want to modify BidiChecker or understand its internal workings). For an example of how to use the precompiled package, see samples/reviews/reviews_packaged_test.html.
  2. a local copy of the bookmarklet installation page ( bookmarklet.html and bookmarklet.css). The bookmarklet links generated by this page will load Javascript files from the same installation directory, allowing you to easily run BidiChecker on a local web server.
  3. The BidiChecker GUI code, located in the gui-app subdirectory. This contains a precompiled file gui_compiled.js, the web pages which host the GUI, and their style file.
Note that the entire lib/ subdirectory tree should be installed on a web server and accessed via http: or https:.

BidiChecker source code uses the Closure Library, an open-source library of useful Javascript modules. If you want to run a modified version of BidiChecker or run BidiChecker's own unit tests, you will need to install a copy of the Closure Library on your system. To do that, run the following command from the same working directory as you ran the above command:

svn checkout http://closure-library.googlecode.com/svn/trunk/ bidichecker/third_party/closure-library

BidiChecker does not currently easily support linking with a copy of the Closure Library at a different location. Please note that you do not need to install the Closure Library to use the precompiled version of BidiChecker.

For an example of how to use BidiChecker with the Closure Library, see samples/reviews/reviews_closure_test.html.

Platform support

The core BidiChecker backend Javascript code is regularly tested on a selection of browsers. The browser bookmarklet has been tested on Firefox and Chrome. It is not currently supported on Internet Explorer prior to version 9.

Regardless, the bidi errors identified by the tool are not platform-specific; the code searches the DOM for errors, and it may find errors which only cause display garbling on browsers other than the one on which it is run. It should not be necessary to run the tool on a particular browser in order to find bidi errors for that browser.
